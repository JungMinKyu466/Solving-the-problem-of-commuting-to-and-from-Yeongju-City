# 🔥 Shall we DATATHON? 🔥

> **SeD 2nd 상반기 프로젝트 집중 개발 세션**
> *from Idea to Prototype*
테스트용
---

## 🧭 개요

2026년 상반기, SeD 2기는 각 팀이 준비해온 공모전·프로젝트 아이디어를 바탕으로
**내부 데이터톤**을 진행합니다.

이번 데이터톤은 단순히 밤새 결과물을 만드는 행사가 아니라,
각 팀의 프로젝트를 **아이디어 수준**에서 **제출 가능한 산출물 수준**으로 끌어올리기 위한
집중 개발 세션입니다.

핵심 목표는 다음과 같습니다.

**문제 정의 → 데이터 정리 → EDA/모델링 → 인사이트 도출 → 발표자료 제작 → 피드백 반영**

즉, 이번 데이터톤은 이후 정기세션 발표와 공모전 제출을 위한
**프로젝트 중간 전환점**입니다.

---

## 👥 참여 팀

이번 데이터톤은 아래 4개 팀이 참여합니다.

* **Team 영천의미학**
* **Team 데이터분석해조**
* **Team InsightLab**
* **Team F4**

각 팀은 현재까지 정리한 주제와 데이터를 바탕으로
분석 방향을 구체화하고, 제출 가능한 형태의 산출물을 제작합니다.

---

## 💡 활동 구성

### 1. 내부 데이터톤

> 📍 장소: 공대 8호관 502호
> ⏰ 진행 방식: 12시간 집중 데이터톤

* **목표**
  팀별 프로젝트를 실제 분석·서비스·정책 제안 산출물로 구체화

* **주요 작업**

  * 문제 정의 보완
  * 데이터셋 정리 및 변수 정의
  * EDA 수행
  * 모델링 또는 분석 방법론 적용
  * 중간 결과 해석
  * 공모전 제출용 발표자료 초안 제작
  * 향후 보완 계획 정리

* **권장 제출물**

  * 발표자료: `SeD_2nd_Datathon_{Teamname}.pptx`
  * 분석 코드: `SeD_2nd_Datathon_{Teamname}.ipynb`
  * 데이터/변수 정의표: `SeD_2nd_Datathon_{Teamname}_DataDictionary.xlsx` 또는 `.csv`
  * 필요 시 추가 산출물: 대시보드, 지도 시각화, 서비스 목업, 정책 제안서 초안 등

## ✅ 필수 포함 항목

각 팀의 발표자료에는 아래 항목이 반드시 포함되어야 합니다.

1. **프로젝트 주제**

   * 해결하려는 문제
   * 왜 중요한 문제인지
   * 공모전 취지와의 연결성

2. **사용 데이터**

   * 데이터 출처
   * 데이터 구조
   * 주요 변수
   * 결측치·좌표·단위 등 현재 이슈

3. **분석 방법**

   * EDA 방식
   * 모델링 또는 시각화 방법
   * 사용한 도구와 라이브러리

4. **현재까지의 결과**

   * 주요 그래프, 지도, 표
   * 중간 인사이트
   * 예상되는 최종 산출물

5. **한계와 다음 단계**

   * 현재 부족한 부분
   * 다음 주까지 보완할 작업
   * 공모전 제출까지 필요한 일정

---

## ⭐ 가점 요소

다음 항목이 포함되면 결과물 완성도가 높아집니다.

* 변수 정의표 또는 데이터 명세서 작성
* 분석 단위 명확화
  예: 격자, 자치구, 집계구, 사업장, 정류장, 상권 등
* 세부 EDA 및 시각화
* 모델 적용 및 성능 검증
* 지도 기반 시각화
* 실제 정책/서비스 활용 시나리오
* Before/After 개선 효과 제시
* 공모전 평가 기준과의 연결
* 코드 재현성 정리
* 역할분담 및 향후 일정표

데이터톤 종료 후 각 팀은 보드 발표를 진행합니다.

* 팀별 발표

각 팀이 다음 정기세션에서 더 나은 결과물을 만들 수 있도록 돕는
**프로젝트 리뷰 과정**입니다.
